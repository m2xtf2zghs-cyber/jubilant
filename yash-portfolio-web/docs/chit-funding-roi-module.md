# Chit Funding ROI Module (CFO + Chief Accountant Design)

## Scope and Guardrails
- Purpose: measure true funding cost and true deployment yield for chit-funded capital using dated cashflows (XIRR).
- Existing loan/interest logic remains untouched.
- This module only consumes tagged outputs (interest/profit/recovery) from existing logic via allocation/return tagging.
- All write operations must be auditable and reconcilable to accounting entries.

- One actual draw event (or versioned events if needed)

- `purpose` (`LENDING|INVENTORY|EXPENSES|ASSET|OTHER`)
- `target_entity_type` (e.g., `LOAN`, `PROJECT`, `VENDOR`)
- `target_entity_id` (string/uuid)
- Optional `linked_loan_id`
- `notes`

### Returns / Recovery to Chit Pool
#### `chit_capital_returns`
- Tracks capital + attributable income flowing back to the chit pool
- `chit_id`, optional `allocation_id`
- `return_date`, `amount_returned`

- `notes`

### Scenarios / Stress Controls

## 3) Calculation Pseudocode (XIRR / Blended IRR)

### A. XIRR helper (dated cashflows)
```text
Input: cashflows[] = [{ date, amount }]
Rules:
- amount > 0 => inflow
- amount < 0 => outflow
- must contain at least one positive and one negative

1. Sort cashflows by date ascending
2. baseDate = first cashflow date
3. For each flow:
   t = days_between(flow.date, baseDate) / 365.0
4. Define XNPV(rate):
   sum(flow.amount / (1 + rate)^t)
5. Solve XNPV(rate) = 0 using:
   - Newton-Raphson (fast path)
   - Bisection fallback on bracket if Newton diverges
6. Return annual effective rate (decimal and %)
```

### B. Chit Effective Cost XIRR (per chit)
```text
cashflows = []
For each actual chit installment payment:
  cashflows.push({ date: payment_date, amount: -amount_paid, type: "INSTALLMENT" })
If chit receipt exists:
  cashflows.push({ date: draw_date, amount: +amount_received, type: "DRAW_RECEIPT" })
  If fees_paid_separately:
    cashflows.push({ date: draw_date, amount: -commission_amount, type: "COMMISSION" })
    cashflows.push({ date: draw_date, amount: -other_charges, type: "OTHER_FEES" })

xirr = XIRR(cashflows)
outputs:
- Chit_XIRR_Annual
- Total_Paid
- Net_Proceeds
- Discount_Cost
- Cost_Rupees_Estimate (cash + rate proxy)
```

### C. Deployment Yield XIRR (per chit, consuming existing interest logic outputs)
```text
yieldFlows = []
For each fund_source_allocation where source_type=CHIT and chit_id = X:
  yieldFlows.push({ date: allocation_date, amount: -amount_allocated, type: "ALLOCATION" })
For each chit_capital_return for chit_id = X:
  inflow = amount_returned + interest_income_amount + other_income_amount
  yieldFlows.push({ date: return_date, amount: +inflow, type: "RETURN_AND_INCOME" })

Yield_XIRR_Annual = XIRR(yieldFlows)
Utilization % = current_deployed / amount_received
Idle Balance = amount_received - net_deployed
```

### D. Net Spread & Profitability
```text
Net_Spread = Yield_XIRR_Annual - Chit_XIRR_Annual
BreakEven_Yield = Chit_XIRR_Annual
Effective_Funding_Cost_Rs = estimate_from_chit_cost_cashflows()
Net_Profit_Rs = Attributed_Income_Rs - Effective_Funding_Cost_Rs

Flag:
  >8% => STRONG_POSITIVE
  3%-8% => MODERATE
  0%-3% => WEAK
  <0% => NEGATIVE
```

### E. Blended Portfolio XIRR across multiple chits
```text
blendedChitFlows = concat(all chit cost cashflows from all chits)
Blended_Chit_XIRR = XIRR(blendedChitFlows)

blendedYieldFlows = concat(all chit-funded allocation/return yield cashflows across chits)
Blended_Yield_XIRR = XIRR(blendedYieldFlows)

Portfolio_Spread = Blended_Yield_XIRR - Blended_Chit_XIRR
```

### F. Monthly Stress Table + Worst-case Simulation
```text
For each month in analysis range:
  installments_due = sum(chit_installments.expected_amount due in month across running chits)
  business_inflows = sum(existing business/lending inflows from ledger/collections in month)
  other_fixed_outflows = user input or config
  net_surplus = business_inflows - installments_due - other_fixed_outflows
  stress_ratio = net_surplus / installments_due (if installments_due > 0)

Flag:
  >=1.5 SAFE
  1.0 to 1.5 WATCH
  <1.0 DANGER

Worst-case scenario:
  inflows *= (1 - reduce_inflows_pct)
  shift inflow dates by delay_collections_days
  apply default impact reduction on collectible inflows
  recompute stress_ratio + yield XIRR
```
